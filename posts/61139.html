Java虚拟机参数

发表于2022-07-27|更新于2022-08-29|参考手册
|字数总计:573|阅读时长:2分钟|阅读量:

JVM参数

内存相关

  1. -Xmx8m 最大堆内存
  2. -Xms8m 最小堆内存
  3. -Xmn 指定新生代大小

GC相关

  1. -verbose:gc // 在控制台打印gc日志
  2. -XX:+PrintGCDetails // 打印gc过程的细节
  3. -XX:+PrintGCDateStamps // 打印gc的发生时间戳
  4. -Xloggc:gc.log // 保存gc日志到文件中，默认在项目的根目录，控制台不再打印
  5. -XX:+PrintGCApplicationStoppedTime // 打印stop world的时间
  6. -XX:+PrintGCApplicationConcurrentTime // 打印程序未中断运行的时间
  7. -XX:+PrintHeapAtGC // 打印gc前后堆栈的信息
  8. -XX:+PrintTenuringDistribution // 打印每次minor gc后新的存活周期的阈值
  9. -XX:+Xnoclassgc // 是否回收类型
  10. -verbose:class // 在控制台打印class日志
  11. -XX:+TraceClass-Loading // 类加载信息
  12. -XX:+TraceClassUnLoading // 类卸载信息，在FastDebug版本的虚拟机中支持该参数

垃圾收集器相关

  1. -XX:SurvivorRatio=x // eden区占年轻代的比例，默认为8，则Eden：Survivor为8：2，其中Survivor区的from和to各占1/10
  2. -XX:PretenureSizeThreshold=x // 大于该值的对象直接在老年代分配
  3. -XX:HandlePromotionFailure=true // 是否设置空间分配担保
  4. -XX:+UseConcMarkSweepGC // 开启CMS垃圾收集器
  5. -XX:ParallelGCThreads // 垃圾收集的线程数
  6. -XX:MaxGCPauseMills=x // Parallel Scavenge垃圾收集器最大垃圾收集停顿时间，G1收集器也可使用
  7. -XX:GCTimeRatio=x // Parallel Scavenge垃圾收集器允许的最大GC时间占比，默认为99，即1/（1+99），允许最大1%的GC时间占比
  8. -XX:+UseAdaptiveSizePolicy // Parallel Scavenge垃圾收集器开启动态调整新生代大小，Eden区和Survivor区比例、晋升老年代对象大小等参数，以提供合适的停顿时间或吞吐量
  9. -XX:CMSInitiatingOccupancyFraction=x // 触发CMS垃圾回收的百分比，默认值为92%
  10. -XX:+UseCMS-CompactAtFullCollection // 用于CMS收集器不得不进行Full GC时开启内存碎片的合并整理过程，默认开启，在JDK9时废弃
  11. -XX:CMSFullGCsBefore-Compaction=x // 要求CMS收集器在执行若干次不整理空间的Full GC之后，下次进入Full GC前必须进行碎片整理
  12. -XX:G1HeapRegionSize=x // 设定G1的region区域大小，取值为1M～32M，为2的N次幂
